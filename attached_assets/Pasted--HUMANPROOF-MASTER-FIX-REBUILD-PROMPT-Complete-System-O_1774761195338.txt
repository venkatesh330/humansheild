### CRITICAL FINDING #1 — Formula weights CONTRADICT the official report

The Technical Report states the correct weights are:
  D1: 30%, D2: 20%, D3: 22%, D4: 18%, D5: 10%

But the live code uses:
  D1: 35%, D2: 25%, D3: 20%, D4: 12%, D5: 8%

**Impact:** D4 (Experience Protection) is underweighted by 6 percentage points.
A 20-year senior surgeon scores as if their experience barely matters.
A senior finance director scores nearly as high as a fresher.
This erodes user trust — "the tool doesn't know that experience matters."

### CRITICAL FINDING #2 — 88% of roles use DEFAULT data (not role-specific)

Out of 322 work type keys in the system:
- D2 TOOL_MATURITY: 283/322 roles (88%) fall back to generic default=52
- D3 AUGMENTATION: 287/322 roles (89%) fall back to generic default=45
- DANGER_SKILLS: 303/322 roles (94%) show the same 4 generic tags
- SAFE_SKILLS: 300/322 roles (93%) show the same 5 generic tags
- TRANSITION_RECS: 0/322 roles — this data structure DOES NOT EXIST IN THE CODE

261 roles have ALL FOUR data points completely defaulted.
Examples: mob_rn, fin_compliance, aero_def, tr_facilitator, ph_manufacturing,
hr_ld, hc_pharmacy, ins_actuarial, gov_social, auto_mfg, saas_pm, sw_embed...

A game designer and a surgeon see IDENTICAL "at-risk skills" and "protected skills."
This is the #1 credibility killer. Users immediately notice this is wrong.

### CRITICAL FINDING #3 — Country Risk formula produces extreme outliers

Current formula: getCountryRisk = d[0] - d[1] + 50
- India [68,28] → score = 90 (adds +7.2pts to every Indian user's final score)
- USA [80,45] → score = 85 (too high — inflates all US scores)
- Bangladesh [45,15] → score = 80

The country dimension should be contextual, not dominant.
Fix: (d[0]-d[1])/2 + 50 → India=70, USA=67.5, Bangladesh=65, Sweden=46.5

### Fix the 5-Dimension Formula

Replace current formula with the correct weights from the Technical Report:

```javascript
// OLD (WRONG): score = d1*0.35 + d2*0.25 + (100-d3)*0.20 + d4*0.12 + d5*0.08
// NEW (CORRECT): 
const rawScore = d1*0.30 + d2*0.20 + (100-d3)*0.22 + d4*0.18 + d5*0.10;
const score = Math.min(98, Math.max(4, Math.round(rawScore)));
```

Weight reasoning to show in UI tooltips:
- D1 30%: Core task automatability — biggest single driver
- D2 20%: Real tools must exist and be production-deployed
- D3 22%: Augmentation is key — if AI makes you 5x more powerful, you're safer
- D4 18%: Experience is STRONGLY protective for most human-judgment roles
- D5 10%: Country context matters but shouldn't dominate individual scores

### Fix Country Risk Formula

```javascript
// OLD: return Math.max(5, Math.min(95, d[0]-d[1]+50))
// NEW (proportional, not explosive):
function getCountryRisk(ck) {
  const d = COUNTRY_DATA[ck] ?? COUNTRY_DATA.other;
  // Divide by 2 to prevent country from dominating the formula
  // Range: 35-75 (not 5-95)
  return Math.max(35, Math.min(75, Math.round((d[0] - d[1]) / 2 + 50)));
}
```

Resulting values:
- India [70,29]: 70
- USA [83,47]: 68
- Sweden [78,86]: 46
- Bangladesh [47,16]: 65
- Singapore [88,73]: 57
