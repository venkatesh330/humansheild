# HumanProof v3.0 — Master Build Prompt

> Copy this entire prompt as your system prompt when building, rebuilding, or extending the HumanProof platform. It incorporates all bug fixes, corrected formulas, architectural upgrades, and market-validated features from the v2.0 audit.

---

## 1. PLATFORM IDENTITY

You are building **HumanProof** — a career AI-resilience assessment platform. It is a React + Vite single-page application that helps workers understand, measure, and improve their resistance to AI-driven job displacement.

**Core promise:** Give any worker a credible, data-backed score of their AI displacement risk and a personalised action plan to improve it.

**Design philosophy:**
- Every number shown must be explainable — show the formula behind every score
- Every risk statement must be actionable — pair every warning with a concrete next step
- Trust is the product — cite every data source, show confidence intervals, never overstate certainty
- Progressive disclosure — tools unlock as the user completes prior assessments

## 4. THE 6-DIMENSION SCORING ENGINE (CORRECTED FORMULAS)

### 4.1 Core formula

```
rawScore = D1×0.26 + D2×0.18 + D3×0.20 + D4×0.16 + D5×0.09 + D6×0.11
```
Weights sum to exactly 1.00. Do not change these without re-running validation.

### 4.4 D3 — Human Amplification (weight: 0.20) — CORRECTED

```javascript
function computeD3(augmentationValue, jobTitle) {
  // Curved inversion: preserves mid-range nuance
  // Exponent 0.70 makes mid-range roles more protected than neutral
  const EXPONENT = 0.70;
  const normaliser = Math.pow(100, EXPONENT); // = 100^0.70 ≈ 25.12
  const d3Raw = 100 - (Math.pow(augmentationValue, EXPONENT) / normaliser) * 100;
  // Example: augVal=50 → d3 = 100 - (50^0.70 / 25.12)*100 ≈ 37 ✓

  // Governance keyword boost
  // Applies ONLY when role explicitly involves governing/auditing AI
  const governanceKeywords = [
    'oversight', 'governance', 'ethics', 'regulation', 'audit',
    'policy', 'accountability', 'compliance'
  ];
  const titleLower = (jobTitle || '').toLowerCase();
  const hasGovernanceKeyword = governanceKeywords.some(k => titleLower.includes(k));

  let d3Final = d3Raw;
  if (hasGovernanceKeyword && augmentationValue > 60) {
    d3Final = Math.min(d3Raw + 8, 85); // +8 boost, hard cap at 85
    // Source: governance role protection rationale (cite specific paper here)
  }

  return Math.max(0, Math.min(100, d3Final));
}
```

### 4.5 D4 — Experience Shield (weight: 0.16) — CORRECTED FLOOR LOGIC

```javascript
function computeD4(baseRoleRisk, yearsOfExperience) {
  // Experience reduces risk proportionally
  const experienceFactors = {
    '0-2':  0.00,
    '2-5':  0.15,
    '5-10': 0.30,
    '10-20':0.50,
    '20+':  0.65,
  };
  const factor = experienceFactors[yearsOfExperience] || 0;
  const reduction = baseRoleRisk * factor;

  // BUG FIX: Floor only applies to HIGH-risk roles
  // Original: always floored at 25% of base — incorrectly inflated safe-role seniors
  // Fixed: floor only activates when base risk > 50
  let d4 = baseRoleRisk - reduction;
  if (baseRoleRisk > 50) {
    const floor = baseRoleRisk * 0.25; // can never reduce by more than 75%
    d4 = Math.max(d4, floor);
  }
  // If base risk ≤ 50, experience can reduce freely (safe role = no artificial floor)

  return Math.max(0, Math.min(100, d4));
}
```

### 4.6 D5 — Country Context (weight: 0.09)

```javascript
function computeD5(countryAdoption, countryRegulation) {
  // adoption: 0–100 (how widely deployed AI is in this country)
  // regulation: 0–100 (how strictly AI is regulated; higher = more protected)
  // Both sourced from OECD AI Policy Observatory data

  // Direction check:
  // High adoption + low regulation = high risk (e.g. USA: adoption~85, reg~30) → D5~75
  // High adoption + high regulation = moderate (e.g. EU: adoption~70, reg~80) → D5~34
  // Low adoption + any regulation = low (e.g. many developing markets) → D5~22–40

  const raw = countryAdoption * (1 - countryRegulation / 100) * 0.80 + 22;

  // Clamp: formula is well-calibrated for OECD data; clamping should be rare
  return Math.max(20, Math.min(85, raw));
}
// NOTE: If raw value > 85 or < 20 more than ~10% of the time,
// recalibrate the 0.80 scalar and +22 offset against updated OECD data.
```

### 4.8 Post-formula adjustments (apply in this exact order)

```javascript
function finaliseScore(rawScore, D1, D2, jobTitle, industry, yearsOfExperience) {
  let score = rawScore;

  // Step 1: Additive D1/D2 interaction boost (CORRECTED — no double-counting)
  // Only activates when BOTH D1 AND D2 are simultaneously very high
  // (emergent risk from AI being both capable AND deployed in this role)
  if (D1 > 70 && D2 > 70) {
    const interactionBoost = (D1 - 70) * (D2 - 70) * 0.001;
    // Example: D1=90, D2=85 → boost = 20*15*0.001 = 0.3 (small, appropriate)
    // Example: D1=95, D2=95 → boost = 25*25*0.001 = 0.625
    score += interactionBoost;
  }
  // REMOVED: original (weighted avg − 50) × 0.15 formula — caused double-counting

  // Step 2: Industry multiplier
  const industryMultipliers = {
    'finance':     1.15,
    'accounting':  1.10,
    'legal':       1.05,
    'technology':  1.00,
    'healthcare':  0.88,
    'education':   0.85,
    'social-work': 0.80,
    // add others...
  };
  score *= (industryMultipliers[industry] || 1.00);

  // Step 3: Specialist title deduction
  const specialistKeywords = [
    'senior', 'lead', 'principal', 'architect', 'chief', 'director',
    'vp', 'head', 'partner', 'fellow', 'distinguished'
  ];
  const titleLower = (jobTitle || '').toLowerCase();
  const isSpecialist = specialistKeywords.some(k => titleLower.includes(k));
  const yearsMultiplier = isSpecialist ? 1.4 : 1.0;
  const experienceDeductions = { '0-2':0, '2-5':3, '5-10':7, '10-20':12, '20+':16 };
  const baseDeduction = (experienceDeductions[yearsOfExperience] || 0) * yearsMultiplier;
  const cappedDeduction = Math.min(baseDeduction, 18); // hard cap
  score -= cappedDeduction;

  // Step 4: Specificity markers
  const specificityKeywords = [
    'custom', 'bespoke', 'embedded', 'niche', 'experimental', 'novel', 'pioneering'
  ];
  if (specificityKeywords.some(k => titleLower.includes(k))) {
    score -= 8;
  }

  // Step 5: Final clamp
  return Math.round(Math.max(3, Math.min(97, score)));
}
```

### 5.2 Scoring formula (corrected weighted mean)

```javascript
function computeHII(dimensionAverages, jobTypeWeights) {
  // dimensionAverages: { emp: 3.4, mor: 4.1, cre: 2.8, phy: 3.9, soc: 3.6, con: 4.0 }
  // jobTypeWeights: { emp: 1.5, mor: 1.0, cre: 1.0, phy: 0.8, soc: 1.2, con: 1.0 }
  // (weights vary by job type — 30 job types have custom weight sets)

  const dims = ['emp', 'mor', 'cre', 'phy', 'soc', 'con'];

  let weightedSum = 0;
  let totalWeight = 0;

  for (const dim of dims) {
    const rawScore = (dimensionAverages[dim] - 1) * 25; // map 1–5 → 0–100
    const weight = jobTypeWeights[dim] || 1.0;
    weightedSum += rawScore * weight;
    totalWeight += weight;
  }

  // CORRECTED: divide by sum of weights, not by 6
  // This ensures weights affect relative importance only, not absolute scale
  const hiiScore = weightedSum / totalWeight;
  return Math.round(Math.max(0, Math.min(100, hiiScore)));
}
```

## 6. DISPLACEMENT FORECAST — CORRECTED S-CURVE

### 6.1 Projection formula (corrected — starts from current score)

```javascript
function projectScore(currentScore, years, decayRate, scenario) {
  // scenario multipliers on the decay rate
  const scenarioMultipliers = {
    optimistic:   0.4,  // user upskills 3+ hrs/week
    baseline:     1.0,  // no change
    pessimistic:  1.8,  // AI adoption accelerates
  };

  const k = decayRate * scenarioMultipliers[scenario];
  const MAX_RISK = 97;

  // Shifted logistic: starts at currentScore, asymptotes toward MAX_RISK
  // NOT the standard logistic starting at 0 — must start at current score
  const projectedScore = currentScore + (MAX_RISK - currentScore) /
    (1 + Math.exp(-k * (years - (MAX_RISK - currentScore) / (2 * k * (MAX_RISK - currentScore)))));

  // Simplified equivalent for implementation:
  // projection(t) = maxRisk - (maxRisk - currentScore) * e^(-k*t)
  // This is a saturating exponential — simpler and equally valid
  const simpleProjection = MAX_RISK - (MAX_RISK - currentScore) * Math.exp(-k * years);

  return Math.round(Math.max(currentScore, Math.min(MAX_RISK, simpleProjection)));
}
```

### 7.1 Custom skill scoring (corrected three-layer logic)

```javascript
function scoreCustomSkill(skillName, userIndustry) {
  const name = skillName.toLowerCase();

  // GOVERNANCE/OVERSIGHT keywords (truly protective regardless of tech context)
  const governanceKeywords = ['ethics', 'governance', 'compliance', 'audit', 'oversight',
    'regulation', 'policy', 'accountability', 'responsible ai'];

  // TECH keywords (not inherently protective — need governance context to be protective)
  const techKeywords = ['ai', 'ml', 'automation', 'llm', 'neural', 'algorithm',
    'machine learning', 'deep learning'];

  // AUTOMATABLE keywords (high risk regardless of how they sound)
  const automatableKeywords = ['invoice', 'data entry', 'processing', 'filing',
    'scheduling', 'transcription', 'copy', 'template'];

  // Layer 1: Automatable check (highest priority — overrides tech-sounding names)
  if (automatableKeywords.some(k => name.includes(k))) return 75 + Math.random() * 15;

  // Layer 2: Truly protective (governance PLUS or WITHOUT tech)
  const hasGovernance = governanceKeywords.some(k => name.includes(k));
  const hasTech = techKeywords.some(k => name.includes(k));

  if (hasGovernance) {
    // "AI Ethics" → protective. "AI Infrastructure" → NOT protective (no governance keyword)
    return 15 + Math.random() * 20; // low risk: 15–35
  }

  // Layer 3: Tech-only skills (risky — AI can replace AI engineers too)
  if (hasTech && !hasGovernance) {
    return 55 + Math.random() * 25; // moderate-high risk: 55–80
  }

  // Layer 4: Industry-aware human-skill keyword matching
  const humanKeywords = ['empathy', 'facilitation', 'mentoring', 'negotiation',
    'coaching', 'leadership', 'strategy', 'storytelling', 'persuasion'];
  if (humanKeywords.some(k => name.includes(k))) return 20 + Math.random() * 20;

  // Default: moderate risk
  return 45 + Math.random() * 20;
}
```

```javascript
function estimateCompletionDate(phase, progressData) {
  const completedThisWeek = Object.values(progressData)
    .filter(p => p.completedAt > Date.now() - 7 * 24 * 60 * 60 * 1000).length;
  const coursesPerWeek = Math.max(completedThisWeek, 0.5); // avoid division by zero
  const remaining = phase.courses.filter(c => !progressData[c.id]?.completed).length;
  const weeksRemaining = Math.ceil(remaining / coursesPerWeek);
  return weeksRemaining;
}
```

## 17. ERROR HANDLING

```javascript
// Each tool tab wrapped in an independent error boundary
class ToolErrorBoundary extends React.Component {
  state = { hasError: false, error: null };
  static getDerivedStateFromError(error) { return { hasError: true, error }; }
  render() {
    if (this.state.hasError) return (
      <ErrorCard
        message="Something went wrong in this tool."
        detail={this.state.error?.message}
        onRetry={() => this.setState({ hasError: false, error: null })}
      />
    );
    return this.props.children;
  }
}
// Each of the 7 tool tabs must be individually wrapped in ToolErrorBoundary
// If one tab crashes, the other 6 continue working normally
```

## 18. VERSIONING & DATA MIGRATION

When you update the scoring model (change weights, formula, or role data):
1. Bump `DATA_VERSION` constant (e.g. `'2026-Q2'`)
2. On app load, detect entries with older versions in `hp_score_history`
3. Show a one-time dismissible banner: "Your score history includes entries from a previous model version. These may not be directly comparable to your latest score."
4. Tag older entries on the drift chart with a visual indicator (e.g. hollow circle vs filled circle)
5. Never silently delete or re-score old entries
